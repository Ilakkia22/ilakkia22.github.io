SQL injection

 

SQL injection is a Web application vulnerability. It occurs when the user input is added  to the sql query without proper sanitization.

Bwapp (Buggy Web application) Solutions:

1.SQL injection(GET/search):

Log in with the default bWAPP credentials. The default credentials are bee/bug. Select the SQL injection(GET/Search) from the dropdown list. This page allows the user to search movies.

                

 

 

Start with a basic SQL payload ‘( single quote). Now we are getting my sql error from the database. So this search option is vulnerable to SQL injection.

                   

By adding a comment(#) at the end of or payload we can prevent execution of remaining SQL query.

For example the original query may be,

 

select title,release,character,genre,imdb from movies where movie_name='<user input>’

select title,release,character,genre,imdb from movies where movie_name=’’#’

 

Anything after the # will be considered as comment.

                   

 

We can perform SQL injection Union attack to determine the number of columns returned by the query.

 

                      

Continue adding column values until the error disappears and the response includes additional content containing the second query values.
